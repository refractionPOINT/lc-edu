Many comparison values support a special "lookback" format. That is, an operation that supports comparing a value to a literal like "system32", can also support a value of <<event/PARENT/FILE_PATH>>. When that value is surrounded by << >>, it will be interpreted as a path within the event and the value at that path will replace the <<...>> value. This allows you to "look back" at the event and use values within for your rule.
