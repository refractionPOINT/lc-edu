The LimaCharlie Data Explorer is a visual way to explore telemetry data for up to a full year. The interactive graph shows process trees and can be navigated in a number of ways. The interface also provides a number of ways to filter available data.

In order to load events into the Data Explorer you need to select a date and time around which to centre your investigation. You can expand the date selector located in the upper left corner of the user interface by clicking on the bar showing the current time stamp. Once the date is selected click on the search icon. The Data Explorer will load event into the users interface and from this state users can navigate across all time.

Logs From Sensor opens a modal from which users can see a list of log files that were collected. This view provides:

  • Time of ingestion
  • Source
  • Log file type
  • Path
  • Retained until date
  • Size

Download Events will a .json file of all events loaded into the browser and make it available for download.

Get All Prevalence inlines additional prevalence data in the event pop-up (covered later in this document).

At the most basic level users can use a simple text filter to reduce the number of data points. The text filtering is done in real-time and applies to all fields. The text filter only applies to data that has been loaded into the user interface.

Using the drop-down menu users can control the broad types of data that are pulled down from the LimaCharlie Cloud. The type of data categories available are as follows.

The Cascading Event Selector is a CSS-like language that allows users to create complex filtering conditions that are applied across the event tree. Using this language users are able to specify a series of space-separated selectors with optional relationship between sets of selectors.

Selector types:

.EVENT_TYPE : selects events of type EVENT_TYPE
#ATOM_ID : selects the event with atom ATOM_ID
@ATOM_ID : selects the event with atom ATOM_ID and all its direct parents as well as children
PLAINTEXT : selects events containing the plaintext PLAINTEXT

A selector beginning and ending with "/" like "/.*_PROCESS/" is interpreted as a regular expression.
Selector relationships:

> : direct child relationship
>> : child relationship across any number of levels

Example : .NEW_PROCESS notepad.exe >> .NEW_PROCESS cmd.exe

Equivalent to: NEW_PROCESS event containing notepad.exe with a child of type NEW_PROCESS containing cmd.exe

Example: ./.*_PROCESS/ system32 > .CODE_IDENTITY c:\\tmp

Equivalent to: a NEW_PROCESS or EXISTING_PROCESS with a direct child event of type CODE_IDENTITY that include the string "c:\tmp"
