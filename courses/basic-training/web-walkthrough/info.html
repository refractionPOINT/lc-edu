User Menu

In the upper left corner of the web application there is a hamburger icon that toggles a slide-out menu which provides access to the following.

Current User simply displays the identity of the current operator.

User Profile opens a modal which provides access to settings for the given account.  From this view users can do the following:

  • Generate an Access Token.
  • Get a copy of the current User ID.
  • Generate User API keys.
  • Toggle the support chat window.

Navigate to the Main Dashboard from anywhere in the application.

Documentation is a direct link to the LimaCharlie documentation.

Dark Theme toggles the application between a light and dark theme.

Log Out signs the operator out of the application.

The root domain and version number are also listed near the bottom.

Access Tokens

The Resource Access Token and the REST endpoint can be used to programatically update resources you've already created.


The token is associated with your user ID since Resources are associated with users and not organizations. The token does not provide access to any other systems.

Clicking the "Generate New Access Token" button will generate a new token, rendering the old one invalid.

Required REST parameters are listed as follows.

uid: your user ID 

access_token: your resource access token

resource_cat: the category of the resource, like "lookup" or "detection" 

resource_name: the name of the resource

[REQUEST BODY]: the body of the POST should contain the data to push, the same when done through the web UI

User API Keys

User API keys are used to generate JWT tokens for the REST API. In contrast to Organization API keys, the User API keys are associated with a specific user (you) and provide the exact same access you have across all organizations you have access to. This makes User API Keys very powerful but also riskier to manage. Therefore we recommend you use Organization API keys whenever possible.


The User API keys can be used through all the same interfaces as the Organization API keys. The only different is how you get the JWT. Instead of giving an OID parameter to https://app.limacharlie.io/jwt, provide it with a UID parameter as specified below.
