Dashboard

When you first login to the web application you will see the main dashboard. This is the highest level view in the user interface and serves as the entry point for managing all aspects of your LimaCharlie account.

In the upper left corner of the dashboard you will find links to the LimaCharlie Terms of Service and Privacy Policy.

The Dashboard is broken down into three main components as defined below.

Groups

Organization Groups allow you to grant permissions to a set of users on a group of organizations. Groups are a top-level user management scheme that allows for the creation of user groups with defined permissions across organizations.

Permissions granted through the group are applied on top of permissions granted at the organization level. Their permissions are additive, and a Group cannot be used to subtract permissions granted at the organization level.

Organization Group Owners are allowed to manage the Organization Group, but are not affected by the permissions. Members are affected by the permissions but do not have the ability to modify the Organization Group.


To create a new Group click on Create Group. After naming the new Group it will be displayed as a line item in the Organization Groups section.


Hovering the cursor over Group Name will display the permissions for that group. Hovering over the number displayed under Organizations, Members or Owners will display the list of the given members to that group.

Clicking on a Group Name  will launch a modal with the Group details. The modal allows users to create and manage all aspects of the Group.


 Recent Activity shows a log of recent activity by members of the group.


Owners contains a list comprised of the Group owners . At least one owner must be present on any given group. To add a user to the group enter their email address into the input field and click Add Owner. To remove an Owner from the group click the X on the right hand side of the user you would like to remove.

Members contains a list that is comprised of Members of the Group. Permission granted through the Group are additive, and a Group cannot be used to subtract permissions granted at the Organization level.


The Organizations section contains a list comprised of the organizations that that the Owners and Members of the Group have access to.


A list of available permissions are as follows.

  • apikey.ctrl: Create, delete and modify API keys.
  • audit.get: Get audit and error messages.
  • billing.ctrl: Change billing information, subscribe to additional services.
  • dr.del: Delete Detection & Response rules in the 'general' namespace.
  • dr.del.managed: Delete Detection & Response rules in the 'managed' namespace.
  • dr.list: List all Detection & Response rules in the 'general' namespace.
  • dr.list.managed: List all Detection & Response rules in the 'managed' namespace.
  • dr.set: Add and update Detection & Response rules in the 'general' namespace.
  • dr.set.managed: Add and update Detection & Response rules in the 'managed' namespace.
  • fp.ctrl: Manage False Positives from detections.
  • ikey.del: Delete installation keys.
  • ikey.list: List all installation keys.
  • ikey.set: Create and update installation keys.
  • incident.get: Get incidents data.
  • incident.set: Set the incidents status.
  • ingestkey.ctrl: Change ingestion keys.
  • insight.det.del: Delete detections from Insight.
  • insight.evt.get: Get event history from Insight.
  • insight.list: List the status of Insight.
  • insight.stat: Get various statistics from Insight.
  • jobs.get: Get jobs data.
  • job.set: Modify the jobs status.
  • module.update: Update and revert sensor version.
  • org.conf.get: Get organization configurations.
  • output.del: Delete Output rules.
  • output.list: List all data Outputs.
  • output.set: Create and update Ouput rules.
  • payload.ctrl: Add/Remove/Get Payloads.
  • payload.use: Deploy a payload to a sensor.
  • replicant.get: Get the list of active Services.
  • replicant.task: Issue requests to Services.
  • sensor.del: Delete a sensor and remove its access.
  • sensor.get: Get information about a sensor.
  • sensor.list: List all sensors.
  • sensor.tag: Add and remove tags.
  • sensor.task: Send commands to a sensor.
  • user.ctrl: Create, delete and change permissions for users.

Member Organizations 

Member Organizations is a list of entry points into the organizations that the operator has access to. Information about each organization is displayed as a line items and each column is explain as follows.

Status: the status column displays a red/green indicator to represent the operational status of the organization.

Name: the assigned name for the given organization.

Errors: displays operational errors such as missing permissions, sensors cloned, outputs failing and many more.

Version: indicates whether the Organization is using the latest version of the sensor.

Online / Quota: shows the ratio of online sensors against the current quota. If the number of sensors online exceeds the number allotted for the quota then an Error will be displayed. Sensors attempting to connect to the LimaCharlie Cloud after the quota 

Description: toggles the visibility of the Organization description entered when it was created.

Active Add-ons (Extensions): displays the number of Active Add-ons and is broken into three categories: API, Detection and Lookups.

My Add-ons (Extensions)

Users of the LimaCharlie platform are able to create their own private extensions or make them available to the public via the Add-ons marketplace.The Add-ons Marketplace is a place where users can subscribe to various capabilities as provided by other users.

To create a new Add-on click on the Create button in the My Add-ons section and fill in the modal form. Creating an Add-on does not immediately grant the Organization access to it. After creating the Add-ons users must go into the Organization and manually subscribe via the Add-ons section. Detailed documentation on User Add-ons can be found here.
