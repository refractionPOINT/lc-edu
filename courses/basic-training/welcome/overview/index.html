LimaCharlie is a powerful set of information security focused capabilities and supporting infrastructure built around a lightweight, cross-platform agent. LimaCharlie provides all of the components required for modern information security operations through a web application or via the API/SDK/CLI. LimaCharlie can be used as a stand-alone threat hunting and incident response platform or be used as a foundation on which to build proprietary security products. 

The LimaCharlie global infrastructure is built on the Google Cloud Platform (GCP) and currently has computing resources available in the USA, Canada, Europe, India and the United Kingdom. Choosing a geographical location ensures data will always be processed in this location and never moved outside. New data centres can be spun up anywhere GCP is available upon customer request.

LimaCharlie employs a multi-tenant architecture. Every cluster is built across multiple zones to maintain high-availability. Clusters are available in multiple regions to deal with any legal requirements and all access to these clusters is transparent. The LimaCharlie Cloud is deployed using Kubernetes and can be horizontally scaled to any size to meet demand dynamically. 

Connectivity between agents and the clusters is done point-to-point and is secured using pinned SSL certificates. This enables easy deployments and the seamless transition of assets between offices and networks.

Each cluster has its own transport keys and each organization has its own command keys. All keys are encrypted in storage and runtime access to the envelope keys is limited to server containers. All sensitive output configurations are encrypted per organization.

Several components work together to provide a set of capabilities that can be tailored to meet the needs of individual users. LimaCharlie is modelled after other cloud service providers, such as Amazon Web Services (AWS) or the Google Cloud Platform (GCP), but designed for the needs of information security operations.

At the bottom of this security stack is a lightweight, cross-platform agent that is installed on the endpoints. The agent (also referred to as a sensor) sends endpoint telemetry back to the cloud where it is processed by the LimaCharlie Detection & Response (D&R) Engine. If a detection occurs the response will be triggered and the given action will be sent back to the endpoint.
