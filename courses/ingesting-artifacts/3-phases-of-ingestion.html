Artifact ingestion and processing involves several steps outlined as follows.

Ingestion

Artifacts are injested into LimaCharlie via the REST API, automatically from the sensor or through the CLI tool.

Normalize to JSON

A copy of the original document is saved and then the file is normailzed into JSON. These normalized JSON files are further indexed across common indicators of compromise.

Detection & Response Engine

The normalized JSON file (the Artifact) and the injestion Event are then run through the LimaCharlie Detection & Response engine where indicators of compromise are checked against detection logic at wire speed.

Custom Retention

The original Artifact and mirror JSON file are then stored for a custom retention period defined by the user.
