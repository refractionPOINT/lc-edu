Interacting with LimaCharlie can always be done via the web app, but day-to-day operations and automation can be done via the Command Line Interface (CLI). Using the CLI will make following this code lab easier.

Install the CLI: pip3 install limacharlie --user

If you don't have pip3 installed, install it. The exact instructions will depend on your Linux distribution.

Create REST API Key

We need to create an API key we can use in the CLI to authenticate with LC. To do so, go to the REST API section of the web app.

  1. In the REST API section, click the "+" button in the top right of the page.
  2. Give your key a name.
  3. For simplicity, click the "Select All" button to enable all permissions. Obviously this would not be recommended in a production environment. Details on LimaCharlie's access scheme can be found here: lc-access
  4. Click the copy-to-clipboard button for the new key and save it somewhere safe as we'll be using it in a future step. Note that the key cannot be retrieved from the web app later.
  5. Back on the REST API page, copy the "Organization ID" at the top of the page and keep note of it as we'll be using it in a future step.

The Organization ID (OID) uniquely identifies your organization and the API key grants specific perissions to this organization.

Log in to the CLI

Back in your terminal, log in with your credentials: limacharlie login

  1. When asked for the Organization ID, paste your OID from the previous step.
  2. When asked for a name for this access, you can leave it blank to set the default credentials.
  3. When asked for the secret API key, enter the key you got from the previous step.
  4. When asked "If this key is a *user* API key", leave the field empty and press Return
  5. You're done! If you issue a limacharlie dr list you should not get any errors.
