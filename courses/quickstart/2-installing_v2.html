Installing the Agent

Before we can start leveraging the capabilities available in LimaCharlie we have to get up and running first. This section takes the user through getting set up with an agent running on an endpoint.

If you have not already, creat an account and your first Organization.

Creating an Account

To gain access to LimaCharlie you will first need to create an account. This account can be attached to a specific email address (which will require you to verify the email), or through a Google, Github or Microsoft account.

Head over to app.limacharlie.io/signup and follow the login instructions. If an email verification is required you will be prompted to do so.

Create an Organization

The first time you login you will be presented with the Terms of Service. Read through them and click accept.

You will be presented with a welcome screen. At the bottom of this screen click "Create Organization". Clicking "Create Organization" opens a dialog to name the Organization and choose the location of the datacenter.

Select a Datacenter

Choose where you want your data to reside. Data does not transit outside of the geographical area in which the data center is located.

The LimaCharlie global infrastructure is built on the Google Cloud Platform (GCP) and currently has computing resources available in the USA, Canada, Europe, India and the United Kingdom. Choosing a geographical location ensures data will always be processed in this location and never moved outside. New data centres can be spun up anywhere GCP is available upon customer request.

Give your organization a name.

The first thing to do is create an Organization. LimaCharlie is multi-tenant and the tenants are structured as Organizations. Sensors belong to Organizations and billing is performed per Organization.

On the right side of the Member Organizations section of the dashboard, click on "Create".

Select a datacenter. This will decide where your sensors will connect to in the world. Many countries are available. Enter an organization name. This name must be unique. The "Name Available" indicator should indicate a green checkbox a few seconds after you've entered a valid name.

LimaCharlie comes with a free tier of two sensors. You do not need a credit card to get this free tier, but if you decide to upgrade your quota, you will have to enter your credit card information.

Click "Create". Within a few seconds you should see an indication that your organization is ready.

Organization View

From the Organizational View navigate to the Install Sensors menu item in the lower portion of the main menu.

Respond

The Respond section contains elements and features that are used for ongoing operations. Here you will find tools that are useful once you have everything set up and running.

  • Sensors

    A top-level listing of all endpoint agents installed. This view allows you to search and drill down into specific endpoints using features that take advantage of LimaCharlie’s real-time connection to the agents.

  • Dashboard

    The Dashboard provides access and insight into several areas of the LimaCharlie platform. Initially there will be nothing to display but once your deployment is up and running this is where you can do the following.

    • Search the entire fleet for all time across common IoC’s (filename, IP, file hash/path, etc).
    • Get top-level information at a glance regarding sensors online over time, as well as events and detections over time.
    • A list of jobs that are currently queued or that have recently run on your deployment.

  • Detections

    The Detections tab is where detections are recorded in list format. From a given Detection the user can go live on the given endpoint or view historical data at the time of detection in the historical data viewer.

  • Artifacts

    The Artifacts tab provides a filterable view of all external artifacts (log files, syslog, PCAPS, Zeek files, ect) that have been ingested by the Organization into LimaCharlie.

  • Search

    The Search tab provides a global search function that allows the user to search across the entire fleet over the past year using common indicators of compromise.

  • Replay

    The Replay tab is where users can run Detection & Response rules against historical telemetry.

Develop

In the Develop section you will find tools and features that get closer to the action.

  • D&R Rules

    The D&R Rules tab is where the current D&R rules as applied to the organization’s feelt reside. False Positive exceptions against these rules are also listed here.

  • Outputs

    The Output tab lists external locations where data is being directed.

  • Exfil Control

    Exfil Control allows users to exercise fine-grained control on the type of data being sent back from the various endpoints.

  • Payloads

    Payloads allows users to upload any executable file to the LImaCharlie cloud from where it can be deployed and run on any set of endpoints.

  • File/Reg Integrity

    The File/Reg Integrity Monitoring tab allows the user to manage all aspects of file and registry integrity monitoring.

  • Yara

    The Yara tab enables the user to manage all aspects of Yara scanning. Add sources, add rules or initiate scans.

  • Artifact Collection

    The Artifact Collection tab enables the user to manage all aspects of external file ingestion for processing by the Detection & Response engine.

  • Net Policies

    The Net Policies tab enables users to create and manage policies for LimaCharlie’s Secure Access Service Edge (SASE).

  • Service Request

    The Service Request tab enables the user to issue requests to specific services manually.

  • Rest API

    The REST API tab is where users can create and manage fine-grained permissions across API keys.

Getting an Installation Key

To get started you will need to create an installation key. Installation keys can be created by visiting the Installation Keys section of the web application and clicking on the plus icon in the upper right corner. 

Once the key has been created copy it to your clipboard. Next visit the Sensor Downloads section of the web application and download the appropriate installer.

Once the sensor is downloaded to the endpoint you will need to give it the appropriate permissions and use the installation key when executing.

Installing the sensor requires administrator (or root) execution:


Windows: installer.exe -i YOUR_INSTALLATION_KEY


MacOS: chmod +x installer ; installer -i YOUR_INSTALLATION_KEY


Linux: chmod +x installer ; installer -d YOUR_INSTALLATION_KEY


Note: On Linux the exact persistence mechanism, like launchd, is left to the administrator, therefore the -d argument launches the sensor from the current working directory without persistence. The sensor does not daemonize itself.

If you wish to remove the sensor from the endpoint you can do so by running the installer with the -c flag.

A sample script for setting up persistence on Linux systems that utilize init.d can be found here: init.d

A sample script for setting up persistence on Linux systems that utilize systemd can be found here: systemd

Once the sensor is installed it should show up in the Sensors section of the web application. To see the Sensor View for a specific Device click the hostname from this list.

Sensor Details

To view the endpoint details click on the hostname in the sensor list on the Sensor page.

This view provides us with a window into the endpoint where we can take a host of actions or perform investigations. A listing of the various menu items, along with a brief description of each is as follows.

Overview

This provides quick access to basic information about the Sensor running on the endpoint. Here you can find the following.

  • Hostname
  • Internal ID
  • Sensor ID
  • Installer ID
  • Enrollment Data
  • Network COnnection Status
  • Platform
  • External IP
  • Organization ID
  • Device ID
  • Last Time Alive

Users can also take the following actions on the endpoint from this view.

  • Isolate the host from the network while leaving a line of control via LimaCharlie.
  • Run a Sweep of the Sensor that searches for common indicators of compromise and returns a host of useful information for beginning an investigation.
  • Delete the Sensor.

Artifacts

This view displays a list of recent artifacts that have been ingested from the given endpoint.

Timeline

The Timeline is a visual data explorer that enables the user to examine up to a year of full telemetry for a given endpoint.

Console

The Console enables users to send any of Sensor Commands to the endpoint and get the results in real-time.

Processes

The Processes view shows a list of all processes as they are running on the endpoint. From this view users can see which processes are signed, listening or talking on the network. Users can also perform the following for any given process.

  • View Modules
  • Kill Process
  • Resume Process
  • Download Memory Strings
  • View Memory Strings

Network

The Network tab shows basic information about network connections in and out of the box. Local IP, local port, foreign IP, state and process hash for every process talking over the network.

File System

The File System tab provides the user with a navigable view of the file system on the endpoint. Users can get a copy of a file hash or download any file.

Live Feed

The Live Feed provides a stream of Events as they are taking place in the endpoint in real-time.
